欢迎收听本期技术小记。今天的故事,是一次测试帮我们抓住的真 bug。

事情是这样的。我们想给 jcode 的两个斜杠命令补端到端测试,一个是回退,一个是看改动差异。本以为是写测试,结果跑起来发现:Rust 后端的差异命令,根本看不到用户的编辑。你明明改了文件,执行查看差异,屏幕上只有一堆 jcode 自己的会话数据库的噪声,真正的改动一行都不显示。而 Python 后端却好好的。

我们一层层往下挖。根因很有意思:检查点是把改动暂存到一个隔离的索引里,用户的编辑从来没进过主索引。而 Rust 那边直接拿 git diff 去比,可 git diff 天生就不显示未跟踪的文件,于是编辑全部隐形。Python 之所以正常,是它在比之前,悄悄把未跟踪文件先登记了一下,让它们浮现出来。

还有第二个毛病:两端都把 jcode 自己的会话目录塞进了快照。Python 本来有排除机制,可它排除的是老名字 letta,而真实目录早就改成了 jcode,排除等于失效。

于是我们两端一起修。Rust 这边用了一个更干净的办法:把当前工作区先打包成一棵临时的树,排除掉会话目录,再拿它去和检查点比。既能显示编辑,又不污染用户的索引,比 Python 的做法还利落。Python 这边,就让它同时排除新旧两个名字。

修完怎么验证?我们用真实的守护进程,配真实的大模型,把跨后端一致性测试跑了一遍。差异命令通过,回退命令也没被带坏,Python 那个共享库八十个测试全绿,格式和静态检查也都干净。

最后全部落地:四个仓库提交推送,守护进程重新编译安装,虚拟环境刷新,还补了一篇架构决策记录。现在你在 jcode 里改完文件看差异,看到的就是你真正的改动,而不再是一堆会话库的噪声了。

这就是今天的故事,我们下期再见。
